# Remote SQLite Database Viewer - Implementation Reference

## Purpose

This document details the SQLite database viewer feature, enabling users to browse and query SQLite databases generated by agents directly in the browser without external tools.

## Feature Overview

When an agent creates a SQLite database file (`.db`, `.sqlite`, `.sqlite3`) in its workspace, users can:
- View database schema (tables, views, columns)
- Browse table data with pagination
- Execute custom SQL queries
- Export query results as CSV

**Flow:**
1. User clicks SQLite file in environment panel → file expands to show tables
2. Frontend fetches table list from backend → backend proxies to agent-env
3. User clicks on a specific table → database viewer opens in new tab with table pre-selected
4. Frontend fetches schema from backend → backend proxies to agent-env
5. Agent-env reads SQLite file, returns schema/query results
6. User browses data in Auto mode (paginated) or writes custom SQL in Manual mode

## Architecture

```
Frontend UI → Backend Proxy → Agent-Env → SQLite File
(React)       (FastAPI)       (FastAPI)    (/app/workspace/)
```

**Two Query Modes:**
- **Auto Mode:** Read-only SQL, auto-generated `SELECT * FROM table LIMIT/OFFSET`, pagination controls
- **Manual Mode:** Editable SQL textarea, execute any query, no pagination

## SQLite Query Execution

**Service Method:** `backend/app/env-templates/python-env-advanced/app/core/server/agent_env_service.py`

### Key Methods

| Method | Purpose |
|--------|---------|
| `is_sqlite_file(filename)` | Check if file has SQLite extension |
| `get_database_tables(path)` | Return list of table/view names |
| `get_database_schema(path)` | Return full schema with PRAGMA table_info |
| `execute_query(path, query, page, page_size, timeout)` | Execute SQL with pagination |

### Query Execution Logic

1. **Path Validation:** Uses `validate_workspace_path()` to prevent directory traversal
2. **Query Type Detection:** Parses first keyword (SELECT, INSERT, UPDATE, DELETE, OTHER)
3. **SELECT Queries:**
   - Wraps in COUNT subquery for total row count
   - Applies LIMIT/OFFSET for pagination
   - Returns columns, rows, pagination metadata
4. **DML Queries:**
   - Executes and commits
   - Returns `rows_affected` count
5. **Error Handling:**
   - Returns error in response (not HTTP error) for frontend display
   - Classifies errors: `syntax_error`, `timeout`, `file_error`, `execution_error`

### Configuration

- **Page Size:** 1000 rows (default)
- **Timeout:** 30 seconds (configurable per request)
- **Extensions:** `.db`, `.sqlite`, `.sqlite3`

## Frontend Implementation

### Route

**File:** `frontend/src/routes/_layout/environment/$envId/database.tsx`

- URL: `/environment/{envId}/database?path={dbPath}&table={tableName}`
- Validates search params with Zod schema
- Renders `DatabaseViewer` component

### Components

**File:** `frontend/src/components/Environment/DatabaseViewer.tsx`

Main component with:
- **Schema Sidebar:** Expandable tables/views with column details
- **Mode Toggle:** Switch between Auto/Manual modes
- **Query Editor:** Textarea (read-only in Auto, editable in Manual)
- **Results Table:** Displays query results with sticky headers
- **Status Bar:** Row count, execution time, pagination controls
- **CSV Export:** Download current results as CSV file

**Key State:**
- `mode`: "auto" | "manual"
- `selectedTable`: Currently selected table name
- `query`: SQL query text (Manual mode)
- `page`: Current page number (Auto mode)
- `expandedSchema`: Set of expanded schema sections

### Tree Integration

**File:** `frontend/src/components/Environment/TreeItemRenderer.tsx`

- Detects SQLite files via `fileType === "sqlite"`
- SQLite files are expandable like folders (click to expand/collapse)
- When expanded, fetches tables from `/database/tables/{path}` API
- Displays tables as children with table icons
- Clicking on a specific table opens database viewer with that table pre-selected
- Uses `window.open()` with encoded path and table parameters

**State Management:** `frontend/src/components/Environment/EnvironmentPanel.tsx`

- `databaseTables` state: `Record<string, { tables: DatabaseTableItem[], loading: boolean, error: string | null }>`
- `handleFetchDatabaseTables(path)`: Fetches and caches table names for a SQLite file
- Tables are fetched on-demand when SQLite file is expanded

**File:** `frontend/src/components/Environment/FileIcon.tsx`

- Database icon (orange) from lucide-react for SQLite files

**File:** `frontend/src/components/Environment/utils.ts`

- `isSQLiteFile(filename)` - Extension-based detection
- `getFileExtension()` - Returns "sqlite" for database files

## Security Considerations

**Path Validation:**
- Reuses existing `validate_workspace_path()` method
- Prevents directory traversal (rejects `..`, absolute paths)
- Validates symlinks don't escape workspace

**Query Execution:**
- All SQL allowed (per requirements)
- Timeout prevents runaway queries
- Each query opens/closes connection (no persistent connections)
- Read-only connection option available if needed

**Access Control:**
- Backend checks user owns the agent
- Environment must be in "running" status
- Agent-env validates auth token on all requests
